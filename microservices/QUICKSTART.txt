⏱️  QUICK START (25 minutes)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: Configure GitHub Secrets (5 minutes)
  ├─ Go to: Repository > Settings > Secrets > New repository secret
  ├─ Add: AWS_ACCESS_KEY_ID
  ├─ Add: AWS_SECRET_ACCESS_KEY
  ├─ Add: GITHUB_TOKEN
  ├─ Add: ARGOCD_TOKEN (see Step 2)
  └─ Add: SLACK_WEBHOOK_URL (optional)
  
  📖 Detailed: See GITHUB-SETUP.md

STEP 2: Setup ArgoCD on EKS (10 minutes)
  ├─ cd microservices
  ├─ chmod +x scripts/setup-argocd.sh
  ├─ ./scripts/setup-argocd.sh aws-project-eks-dev eu-west-1
  ├─ Copy ArgoCD URL and password
  ├─ Generate token: argocd account generate-token
  └─ Add ARGOCD_TOKEN to GitHub secrets (Step 1)

STEP 3: Deploy Microservices (10 minutes - Choose ONE)
  
  Option A: HELM (Direct, Fastest) ⭐
    cd microservices
    ./scripts/helm-deploy.sh dev latest
  
  Option B: Ansible (Infrastructure as Code)
    pip install -r microservices/requirements.txt
    ./scripts/ansible-deploy.sh dev latest
  
  Option C: ArgoCD (GitOps, RECOMMENDED) ⭐⭐
    ./scripts/argocd-deploy.sh dev latest
  
  Option D: GitHub Actions (Automatic)
    git push origin main
    Monitor: Repository > Actions

STEP 4: Verify Deployment (5 minutes)
  ├─ kubectl get pods -n microservices
  ├─ kubectl get svc -n microservices
  ├─ API_IP=$(kubectl get svc api-gateway -n microservices \
      -o jsonpath='{.status.loadBalancer.ingress[0].hostname}')
  └─ curl http://$API_IP/health
